Rule #1: Every feature you don't build is a bug you can't have. Your only goal for v1 is to let two people play Rock Paper Scissors — against a bot, or against each other using a room code. That's it. Everything else is v2 and beyond.
Why so few screens? Most first apps fail because the developer tries to build everything at once, burns out, and ships nothing. 10 screens is completable. The design you already have is beautiful — now you need working code in people's hands.

⚠️ ERROR STATES YOU MUST BUILD

These are not optional. Every one of these will happen to real users. Design and implement them before calling v1 done.

Error Screen Priority What to show
Wrong password Sign In MUST HAVE Red border on password field. Text: "Wrong password. Try again." Don't clear the username field.
Username already taken Sign Up MUST HAVE Red border on username field. Text: "Username taken. Choose another."
Empty fields on submit Sign In / Sign Up MUST HAVE Red border on the empty field. Text: "This field is required." Shake animation helps.
No internet Any MUST HAVE A pixel-art popup: WiFi icon with X, message, RETRY button. Required by app stores.
Invalid room code Room Code MUST HAVE Text below input: "Code not found. Check and try again." The code might be expired or mistyped.
Room code expired Room Code MUST HAVE Text: "Code expired. Create a new one." with a Generate New Code button.
Opponent disconnected mid-match Game Screen MUST HAVE Popup: "Opponent left the match." with just a Main Menu button. Don't give a win by default for v1 — it's complex to cheat-proof.
Lost connection mid-match Game Screen MUST HAVE "Reconnecting…" overlay. If fails after 10s: "Connection lost" popup with Main Menu button.
Server error (500) Any API call MUST HAVE Generic pixel popup: "Something went wrong. Try again." OK button. Log the error silently.
Confirm logout Settings SHOULD HAVE A pixel dialog: "Are you sure you want to log out?" with NO and YES buttons. Prevents accidental logout.
Session expired Any authenticated screen SHOULD HAVE Auto-redirect to Sign In with a small message: "Session expired. Please sign in again."
Profanity in username Sign Up SHOULD HAVE Text: "Username not allowed. Please choose another." Never reveal the blocklist.
Build these as reusable components: A PixelDialog component (error message + one or two buttons) and a FieldError component (red text below an input) cover 90% of these cases. Build them once, use everywhere.

The Only API Endpoints You Need

POST/auth/signupCreate account, return JWTNo

The WebSocket events handle all the multiplayer — no extra REST endpoints needed for VS Friend mode.

Key Rules to Follow

  • Never store passwords as plain text. Always bcrypt hash them on the server.
  • All game logic for VS Friend mode runs on the server. Never trust the client's claimed winner.
  • Bot game logic runs entirely on the client. No server call until the match is finished.
  • Test on a real physical Android device every week. The simulator lies about performance.
  • Build the error states at the same time as the happy path — not after.
  • Use EAS Build from day one. Building locally for iOS requires a Mac with Xcode.
  • Set up Sentry before your first real user. You need crash reports from the start.
  • Room codes expire after 5 minutes in Redis. Always show a countdown on the room code screen.
  • Add a 10-second move timeout in multiplayer — if a player doesn't pick, auto-pick a random move.
  • Ship Phase 2 (VS Bot) to TestFlight and let friends test it before starting Phase 3.
